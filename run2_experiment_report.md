# run2.sh 实验设计说明报告

## 1. 报告目的

这份报告用于解释根目录 `run2.sh` 脚本到底做了什么、整个实验流程是什么、20 组实验分别在测什么，以及实验最终会产出哪些结果文件。

这是一份**实验设计报告**，不是结果报告。当前实验结果尚未跑出，因此本文重点解释：

1. 为什么要做这批实验
2. 这批实验的固定设置是什么
3. 这 20 组超参数组合分别在探索什么
4. 脚本每一步在做什么
5. 最终会输出哪些日志和指标

---

## 2. 实验目标

本实验的核心目标是：

- 在 **FaceScrub** 数据集上，系统评估根目录项目中的 **Slot Attention + Cross Attention 正则化架构**
- 在保持主干网络、切分位置、瓶颈结构和攻击流程基本一致的前提下，通过超参数搜索找到**比原始 baseline 更优**的配置
- 同时关注两类指标：
  - **任务效用（utility）**：分类精度是否足够高
  - **隐私保护（privacy）**：模型反演攻击是否更难

换句话说，这个实验不是单纯追求最高准确率，也不是单纯追求最难重建，而是在寻找一个更优的 **效用-隐私折中点**。

---

## 3. 当前实验到底是什么任务

### 3.1 任务形式

这不是标准的端到端分类训练，而是一个 **split learning / collaborative inference** 场景：

- 客户端侧模型先把输入图像编码成中间表示（smashed data）
- 服务器侧模型继续完成分类
- 攻击者尝试从中间表示反推出原始图像

因此整个实验分成两段：

1. **训练分类模型**
2. **训练并评估模型反演攻击器**

最终要一起看：

- 分类模型的最佳验证精度
- 模型反演攻击的重建质量

### 3.2 使用的数据集

- 数据集：`FaceScrub`
- 输入分辨率：`64x64`

选择 FaceScrub 的原因是：

- 人脸图像比 CIFAR-10 更复杂
- 更适合检验 slot-based 结构化表示是否真正有效
- 更适合观察高保真图像上的隐私泄露风险

---

## 4. 脚本整体流程

`run2.sh` 的执行流程可以概括为：

1. 在根目录创建本次运行目录 `run2log_[时间]`
2. 创建总控制台日志、实验清单、汇总 CSV
3. 定义 20 组超参数
4. 逐组运行实验
5. 每组实验先训练模型，再运行模型反演攻击
6. 每组实验结束后自动解析日志，提取关键指标写入汇总 CSV
7. 全部实验结束后，保留完整日志与总结表，便于后续下载分析

可以简单理解为：

`run2.sh = 一个自动化超参数搜索器 + 一个日志归档器 + 一个结果汇总器`

---

## 5. 脚本中的固定设置

为了保证实验可比性，很多核心配置是固定不变的。

### 5.1 固定不变的模型骨架

- 架构：`vgg11_bn_sgm`
- 切分层：`cutlayer=4`
- 客户端数：`num_client=1`
- 数据集：`facescrub`
- 协同推理方案：`V2_epoch`
- 瓶颈：`noRELU_C8S1`

这意味着所有 20 组实验共享同一条基本主线：

- 相同 backbone
- 相同 split 位置
- 相同 smashed-data 通道数
- 相同 FaceScrub 输入任务

因此 20 组实验的差别主要来自**训练强度、注意力强度、隐私正则强度**，而不是来自模型骨架变化。

### 5.2 固定不变的防御框架

- `AT_regularization = SCA_new`
- `gan_loss_type = SSIM`
- 训练期 decoder 类型：`res_normN4C64`
- 攻击期 decoder 类型：`res_normN8C64`

含义是：

- 训练主线保持现有的注意力-CEM框架
- 攻击评估使用固定的 decoder 结构，确保不同实验之间可比

### 5.3 固定不变的攻击评估设置

- 攻击方式：`MIA`
- 攻击训练轮数：`50`
- 平均次数：`1`

这样做的目的是：

- 把主要变化集中在防御模型本身
- 不把攻击器预算变化混入比较结论

## 6. 这个脚本里的循环到底在干什么

脚本中真正的“大循环”只有一层：**遍历 20 组实验配置**。

每一轮循环会做以下事情：

### 第一步：读取一组超参数

脚本中的 `EXPERIMENTS` 数组一共有 20 条记录，每条记录的格式是：

`batch_size | lr | epochs | noise | lambd | var_threshold | slots | heads | iterations | attention_scale | warmup`

也就是说，每一次循环都会取出一组完整超参数。

### 第二步：生成本实验的唯一名字

脚本会自动为这一组实验生成一个唯一的 `filename`，把所有关键超参数写进文件名中。

这样做的作用是：

- 不同实验不会互相覆盖
- 看到目录名就知道这组实验用了什么参数

### 第三步：创建本实验的独立日志目录

每组实验都会在：

`run2log_[时间]/experiments/expXX/`

下面生成：

- `experiment.log`：本组实验的完整 shell 级日志
- `params.txt`：本组实验的参数文件

因此你后续下载整个 `run2log_[时间]` 目录后，可以逐组查看每个实验到底发生了什么。

### 第四步：训练分类模型

脚本会调用：

`main_MIA.py`

去训练这组参数对应的 split model。

这一阶段会输出：

- 训练过程日志
- 验证精度
- 最佳 checkpoint

### 第五步：运行模型反演攻击

分类模型训练结束后，脚本会继续调用：

`main_test_MIA.py`

读取训练好的模型，并对其进行模型反演攻击评估。

这一阶段会输出：

- 训练时攻击（training-time attack）指标
- 推理时攻击（inference-time attack）指标

### 第六步：自动提取关键指标

脚本会自动解析：

- `MIA.log`
- `MIA_attack_0_0.log`

从中提取：

- 最佳验证精度
- 训练时 MIA 的 `MSE / SSIM / PSNR`
- 推理时 MIA 的 `MSE / SSIM / PSNR`

并写入最终汇总表 `run2_summary.csv`。

所以这 20 次循环，不是“20 次相同重复实验”，而是：

**20 次不同超参数配置的完整训练 + 完整攻击评估**

## 7. 20 组实验在测什么

这 20 组实验并不是随机乱排，而是按强度逐步增强来设计的。

总体思路是：

- 前几组偏保守，优先保 utility
- 中间几组追求 utility / privacy 平衡
- 后几组更激进，测试强隐私区间

### 7.1 第一阶段：低强度探索（实验 1-4）

目的：

- 先看较弱正则下，模型能不能保持较高分类精度
- 观察少量 slot、较小 warmup、较短迭代时的基础表现

特点：

- 较小噪声
- 较小 `lambd`
- 较低 `var_threshold`
- 较少 slots
- 较低 attention scale

### 7.2 第二阶段：中低强度平衡（实验 5-8）

目的：

- 在精度和隐私之间寻找第一个可能的平衡区间

特点：

- batch size 提高到 `256`
- `lambd` 增强
- `slots` 增加到 `8-10`
- 训练轮数提升到 `300`

### 7.3 第三阶段：主力平衡搜索（实验 9-12）

目的：

- 这是最值得重点关注的一段
- 预计最有可能出现“比 baseline 更优”的候选点

特点：

- 训练轮数提升到 `320`
- `var_threshold=0.15`
- `iterations=5`
- `attention_scale` 明显增强

### 7.4 第四阶段：强隐私探索（实验 13-16）

目的：

- 测试在更强正则、更高 slot 数、更长 warmup 下，是否还能守住 utility

特点：

- batch size 达到 `288`
- 噪声更强
- `slots=10-14`
- `attention_scale` 更高

### 7.5 第五阶段：激进强度区间（实验 17-20）

目的：

- 检验注意力正则在非常强的隐私压力下还能否稳定
- 观察高强度配置是否会出现明显的精度崩溃

特点：

- epoch 拉到 `340`
- `iterations=6`
- warmup 达到 `16-18`
- `attention_scale` 接近 `1.0`
- `lambd` 和噪声都更高

## 9. 训练阶段到底做了什么

训练阶段由 `main_MIA.py` 调起。

### 9.1 输入

脚本把以下关键参数传给训练程序：

- 模型结构参数：`arch / cutlayer / bottleneck_option`
- 数据参数：`dataset / batch_size`
- 优化参数：`learning_rate / num_epochs / local_lr`
- 隐私正则参数：`regularization_strength / lambd / var_threshold`
- 注意力参数：`num_slots / num_heads / num_iterations / attention_loss_scale / attention_warmup_epochs`

### 9.2 训练逻辑

训练程序会：

1. 构建 split model
2. 在客户端侧生成 smashed data
3. 在服务器侧继续分类
4. 计算分类损失
5. 在 warmup 之后逐步启用 attention-based robustness regularizer
6. 每个 epoch 做验证
7. 保存最佳模型

换句话说，训练阶段真正优化的是：

- **主任务分类能力**
- **中间表示的隐私鲁棒性**

---

## 10. 攻击阶段到底做了什么

攻击阶段由 `main_test_MIA.py` 调起。

### 10.1 攻击阶段的目的

训练完分类模型后，不能只看准确率，还要验证：

> 攻击者能不能从 smashed data 重建出原始人脸？

### 10.2 攻击阶段流程

攻击程序会：

1. 读取训练好的最佳模型 checkpoint
2. 构造一个 decoder / inversion model
3. 用中间表示去重建原图
4. 分别统计：
   - training-time attack 指标
   - inference-time attack 指标

### 10.3 输出指标

最终会记录三个核心重建指标：

- `MSE`
- `SSIM`
- `PSNR`

通常可做如下直观理解：

- `MSE` 越大，重建越差，隐私通常越强
- `SSIM` 越小，重建结构越差，隐私通常越强
- `PSNR` 越小，重建质量越差，隐私通常越强

因此最终评价不是单看精度，而是：

**在精度尽量高的情况下，是否让重建攻击更难。**

## 11. 这次实验最终会产出什么文件

每次运行 `run2.sh` 后，根目录会自动生成：

`run2log_[timestamp]/`

其中包括：

### 11.1 总体文件

- `run2_console.txt`

- `run2_manifest.tsv`
  - 每组实验的状态清单
- `run2_summary.csv`
  - 每组实验的最终指标汇总表

### 11.2 每组实验的独立文件

在：

`run2log_[timestamp]/experiments/expXX/`

下面会有：

- `experiment.log`
  - 该组实验完整运行过程
- `params.txt`
  - 该组实验完整参数配置

### 11.3 模型与程序原生日志

在：

`run2log_[timestamp]/saves/<experiment_name>/`

下面会有：

- `MIA.log`
  - 训练日志
- `MIA_attack_0_0.log`
  - 攻击日志
- checkpoint 等模型文件

## 12. 组会上可以怎么讲这套实验

可以按下面的逻辑讲：

### 12.1 一句话版本

> 我们在 FaceScrub 上设计了 20 组系统超参数实验，评估基于 Slot Attention 的 split-learning 隐私正则化方法，目标是在分类精度和模型反演攻击鲁棒性之间找到比原始 baseline 更优的折中点。

### 12.2 三步讲解版本

**第一步：为什么做这个实验**

- CIFAR-10 太简单，难以体现高保真人脸上的隐私风险
- FaceScrub 更适合评估结构化人脸表示与重建攻击对抗
- 我们希望验证 slot-based 中间表示是否比传统方式更适合高分辨率人脸场景

**第二步：实验怎么做**

- 固定 backbone、split layer、bottleneck 和攻击方式
- 系统搜索 20 组 attention / regularization 超参数
- 每组都完整经历“训练分类模型 + 运行模型反演攻击”

**第三步：最终看什么**

- 看最佳验证精度
- 看 MIA 的 MSE / SSIM / PSNR
- 找到同时兼顾 utility 和 privacy 的最佳实验点

## 14. 预期分析方式

等结果跑完后，建议用 `run2_summary.csv` 做三类分析：

### 14.1 最佳精度实验

找出 `best_val_acc` 最高的实验，确认 utility 上限。

### 14.2 最强隐私实验

找出：

- `infer_mse` 最大
- `infer_ssim` 最小
- `infer_psnr` 最小

的实验，确认隐私上限。

### 14.3 最佳折中实验

从中挑出：

- 精度没有显著下降
- 但攻击重建质量明显变差

的实验，作为最终主推候选。

## 15. 结论

这份 `run2.sh` 脚本本质上完成了三件事：

1. **自动化超参数搜索**：一次性跑 20 组不同的 attention / privacy 配置
2. **自动化攻击评估**：每组训练后立即进行模型反演攻击
3. **自动化日志与结果汇总**：所有过程和结果自动归档，方便后续汇报和分析

因此，这个实验不是“简单地多跑几次模型”，而是一个面向组会和论文分析都可直接使用的、结构完整的**系统性实验框架**。
