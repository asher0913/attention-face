**项目名称：** FaceScrub 人脸数据集上的模型逆向攻击防御
**实验脚本：** `run3.sh`（超参数搜索，共 20 组实验）
**实验目标：** 超越原始论文 SOTA 防御结果（MIA Inference SSIM ≈ 0.532，越低隐私越好）
**撰写日期：** 2026-03-15

---

## 一、研究背景

### 1.1 分割学习（Split Learning）

分割学习是一种联邦学习变体，将神经网络在客户端（用户设备）和服务器之间进行"切分"：

```
客户端设备                         服务器
┌─────────────────────┐           ┌──────────────────────────┐
│  原始图像 x          │           │  f_tail（后半段网络）      │
│      ↓              │           │      ↓                   │
│  本地编码器 f(·)     │  --z-->   │  分类器                  │
│  （只保留本地）       │  smashed  │  （计算损失、反传梯度）   │
└─────────────────────┘   data    └──────────────────────────┘
```

客户端只向服务器发送"压碎数据"（smashed data）$z = f(x)$，而不发送原始图像，**理论上保护了用户隐私**。

### 1.2 模型逆向攻击（Model Inversion Attack, MIA）

敌手在不接触原始图像的情况下，仅凭截获的 $z$ 训练一个解码器 $D: z \rightarrow \hat{x}$，尝试重建原始人脸图像：

```
攻击者截获 z  →  训练解码器 D  →  重建图像 x̂  →  与原图对比
```

**评估指标（越低表示隐私保护越好）：**

| 指标 | 含义 |
|------|------|
| MSE ↓ | 均方误差，重建图像与原图的像素差异 |
| SSIM ↓ | 结构相似度，感知层面的重建质量 |
| PSNR ↓ | 峰值信噪比，综合图像质量指标 |

**基准线（原始论文 SOTA）：** Inference SSIM ≈ 0.532

## 二、防御机制：CEM（Cluster Entropy Maximization）

### 2.1 核心思想

CEM 的目标是让同一身份的不同人脸图像，其 smashed data $z$ 的**组内方差尽可能大**。
这样即使攻击者得到了 $z$，也无法找到 $x$ 与 $z$ 之间的稳定映射关系，从而破坏重建。

```
同一个人的不同照片   →  理想情况下 z 应该分布很散，而不是聚集在一起
```

### 2.2 CEM 损失函数

$$\mathcal{L}_{\text{CEM}} = \text{ReLU}\left(\log(\text{threshold}) - \log(\text{var per cluster} + \varepsilon)\right)$$

**含义：** 如果某个类别的 smashed data 方差低于阈值，就产生惩罚；否则损失为零。

**阈值计算：**
$$\text{threshold} = \text{var\_threshold} \times \text{noise}^2 + \varepsilon$$

其中 noise 是注入 smashed data 的高斯噪声标准差（$z_n = z + \mathcal{N}(0, \sigma^2)$）。

### 2.3 原始方法（CEM-main）的局限性

原始论文（CEM-main 目录）使用**硬性 KMeans++ 聚类**：

- 每个 epoch 结束后，对全数据集做一次 KMeans 聚类
- 聚类是离散的、不可微的，无法直接进行梯度优化
- 计算代价高，且无法让聚类过程参与端到端训练

## 三、本文创新：SlotCrossAttention-CEM（SCA\_new）

### 3.1 核心改进：用软聚类替换硬聚类

本工作提出 `SlotCrossAttentionCEM` 模块，将 KMeans 替换为**可微分的注意力机制聚类**：

```
输入 z（smashed data，形状 [B, C]）
        ↓
┌─────────────────────────────────────────────────────┐
│  Slot Attention（GRU 迭代竞争）                      │
│  - 初始化 S 个"原型向量"（slots）                    │
│  - 每轮迭代：slots 通过 attention 从 z 中聚合信息    │
│  - K 次迭代后，每个 slot 代表一个软聚类中心          │
└─────────────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────────────┐
│  Cross Attention（tanh 门控精细化）                  │
│  - 以 slots 为 Query，原始 z 为 Key/Value            │
│  - 对每个 slot 进行精细的特征聚合                   │
└─────────────────────────────────────────────────────┘
        ↓
  head_mu / head_logvar（每个 slot 的均值和对数方差）
        ↓
  CEM Loss（基于方差的惩罚）
```

### 3.2 双反向传播训练（Double Backward）

CEM 损失和分类损失需要分开处理梯度：

```python
# Step 1: 计算 CEM 损失，保存编码器梯度
rob_loss.backward(retain_graph=True)
encoder_gradients = {name: param.grad.clone() for name in encoder_params}

# Step 2: 计算分类损失，正常反传
ce_loss.backward()

# Step 3: 手动叠加 CEM 梯度（自适应缩放）
for name, param in encoder_params:
    if lr > 0.00041:
        scale = 0.001 / lr   # 训练早期：CEM 强度弱，优先收敛
    else:
        scale = 1.0           # 训练后期：CEM 全强度
    param.grad += lambd * encoder_gradients[name] * scale
```

**自适应 CEM 强度：** 学习率由 CosineAnnealing 调度，早期学习率大，CEM 强度被自动削弱，让模型先学会分类；后期学习率小，CEM 全力施压以保护隐私。

### 3.3 整体训练流程

```
每个 epoch：
  1. 前向传播：x → f(x) = z  → f_tail(z) → 分类预测
  2. 注入高斯噪声：z_n = z + N(0, σ²)
  3. 将 z_n 送入 SlotCrossAttentionCEM，计算每类方差
  4. 计算 CEM 损失（方差不够大则惩罚）
  5. 双反向传播：分类损失 + 自适应 CEM 梯度叠加
  6. 更新所有参数
```

## 四、实验设计：run3.sh 超参搜索

### 4.1 实验动机

基准实验（`run_exp1.sh`）的超参设置存在以下问题：

| 参数 | 基准值 | 问题 |
|------|--------|------|
| `lambd` | 16 | CEM 梯度权重偏低 |
| `loss_scale` | 0.5 | 相当于 CEM 强度再打 5 折 |
| **实际有效强度** | **16 × 0.5 = 8.0** | **过弱，防御效果有限** |

**实际有效 CEM 强度** = `lambd × loss_scale`（后期训练，学习率充分衰减后）

### 4.2 固定参数（所有实验共用）

| 参数 | 值 | 含义 |
|------|----|------|
| `arch` | vgg11_bn_sgm | VGG11+BN+Sigmoid，带 LCA 稀疏编码层 |
| `cutlayer` | 4 | 网络在第 4 层切分 |
| `bottleneck` | noRELU_C8S1 | 8 通道瓶颈层，无 ReLU |
| `dataset` | FaceScrub | 人脸识别数据集，530 个身份 |
| `batch_size` | 256 | |
| `regularization` | Gaussian\_kl | 高斯 KL 正则化 |
| `AT_regularization` | SCA\_new | 本文提出的注意力 CEM |
| `train_AE` | res\_normN4C64 | 训练时攻击者用 4 层 ResBlock 解码器 |
| `test_AE` | res\_normN8C64 | 测试时攻击者用更强的 8 层解码器 |
| `attack_epochs` | 50 | 攻击者解码器训练轮数 |

### 4.3 搜索的超参数及其含义

| 超参数 | 含义 | 搜索范围 |
|--------|------|---------|
| `lambd` | CEM 梯度权重（越大 = 隐私惩罚越强） | 24 ~ 48 |
| `noise` (σ) | 注入 smashed data 的高斯噪声标准差 | 0.05 ~ 0.10 |
| `var_threshold` | 方差阈值系数（threshold = vt × σ²） | 0.15 ~ 0.25 |
| `loss_scale` | CEM 损失的缩放系数 | 0.8 ~ 1.2 |
| `slots` | Slot Attention 的槽（原型）数量 | 4 ~ 16 |
| `heads` | Cross Attention 的多头数 | 2 ~ 4 |
| `iters` | Slot Attention GRU 迭代次数 | 3 ~ 5 |
| `warmup` | 激活 CEM 前的预热轮数 | 1 ~ 5 |
| `epochs` | 总训练轮数 | 300 ~ 360 |

### 4.4 20 组实验详细配置

实验按 6 个研究问题分组：

---

#### A 组（exp01-03）：基准 → 强化 CEM 梯度强度

> **研究问题：** 单纯提高 `lambd` 和 `loss_scale` 是否改善防御？

| 实验 | lambd | noise | var\_thr | loss\_scale | 有效强度 |
|------|-------|-------|---------|------------|---------|
| **基准** | 16 | 0.05 | 0.15 | 0.5 | **8.0** |
| exp01 | 24 | 0.05 | 0.15 | 1.0 | 24.0 |

**逻辑：** 基准的有效强度仅 8.0，实验将其提升至 24~38，验证 CEM 梯度强度的上限。

---

#### B 组（exp04-07）：提高噪声 + 方差阈值

> **研究问题：** 更高的噪声注入和更严格的方差要求能否进一步扰乱 MIA？

| 实验 | lambd | noise | var\_thr | loss\_scale | 阈值（vt×σ²） |
|------|-------|-------|---------|------------|-------------|
| exp04 | 24 | 0.08 | 0.20 | 1.0 | 1.28e-3 |

| exp07 | 32 | 0.10 | 0.20 | 0.8 | 2.0e-3 |

**逻辑：** 更高噪声 → smashed data 本身更难重建；更高阈值 → 要求方差必须更大，防御门槛更严。

---

#### C 组（exp08-11）：SlotAttention 架构搜索

> **研究问题：** 槽数量（slots）和 GRU 迭代次数对软聚类质量有多大影响？

| 实验 | slots | heads | iters | 变化说明 |
|------|-------|-------|-------|---------|
| exp08 | **16** | 4 | 3 | 更多原型（基准 8 → 16） |
| exp09 | **4** | 4 | 3 | 更少原型（验证下限） |

**逻辑：** slots 越多，聚类粒度越细；iters 越多，注意力迭代越精细。但过多可能造成过拟合或训练不稳定。

#### D 组（exp12-14）：预热（warmup）敏感性

## 六、技术架构细节

### 6.1 VGG11-BN-SGM 主干网络

```
输入图像 (3×64×64)
    ↓ VGG Block 1-4（本地客户端部分）
    ↓ 瓶颈层：Conv(→8ch) + LCALayer（稀疏编码） + Sigmoid
    ─────────── 切分点（cutlayer=4）───────────
    ↓ VGG Block 5-8（服务器 f_tail）
    ↓ 全连接分类器（530类）
```

**LCALayer（局部竞争算法）：** 实现稀疏编码，通过侧抑制机制使激活值更稀疏，本身也有隐私保护效果。

### 6.2 MIA 攻击解码器

```
smashed data z（8×8×8）
    ↓ N 个 ResBlock（残差卷积块）
    ↓ ConvTranspose（逐步上采样）
    ↓ Sigmoid
重建图像 x̂（3×64×64）
```

- 训练阶段评估：`res_normN4C64`（4层 ResBlock）
- **推理阶段评估：`res_normN8C64`（8层 ResBlock，更强攻击者）**

## 七、对比分析

### 7.1 本文方法 vs 原始 CEM 的核心差异

| 维度 | 原始 CEM（CEM-main） | 本文 SCA\_new |
|------|---------------------|--------------|
| 聚类方式 | 硬性 KMeans++ | 软性 SlotAttention（可微分） |
| 梯度流 | 不可微，无梯度 | 端到端梯度优化 |
| 计算时机 | 每 epoch 全数据集 | 每 batch 实时计算 |
| 聚类数 | 类别数（530） | 超参 slots（4~16） |
| 参数量 | 无额外参数 | 少量注意力参数 |

### 7.2 超参搜索的核心假设

1. **有效 CEM 强度不足是基准失败的主因：** 基准 8.0 → 本次实验最高 43.2
2. **更高噪声注入 + 更严格阈值应协同工作：** 噪声保护传输，阈值要求特征更发散
3. **合适的 slot 数量存在最优解：** 过少 → 聚类粒度不够；过多 → 过度分散，损害分类性能
