## 第4页：Split Learning 背景

先介绍一下背景。Split Learning 就是把模型切成两半，前面几层放在客户端作为 encoder，后面的层放在服务器。客户端把输入图片 x 经过 encoder 得到一个中间表示 z，也叫 smashed data，然后把 z 发给服务器继续计算。

问题在于，攻击者可以截获这个 z，然后训练一个 decoder 去重建原始图片。这就是 Model Inversion Attack，简称 MIA。

防御方法有很多种：加高斯噪声、距离相关损失（Nopeek）、瓶颈层压缩通道数、对抗表示学习（ARL）等等。而 CEM 是这篇论文提出的方法，核心思想是最大化条件熵。CEM 的一个很大的优势是，它可以作为插件叠加到上面任何一种防御策略上面，consistently 提升隐私保护效果。

---

## 第5页：CEM 理论

然后讲一下 CEM 的理论基础。论文的 Theorem 1 证明了，攻击者的重建 MSE 有一个下界，这个下界跟条件熵 H(x|z) 成指数关系。所以如果我们能最大化 H(x|z)，攻击者的重建误差就一定会增大，隐私就更安全。

那怎么最大化条件熵呢？论文用的是 GMM——高斯混合模型。每个 batch，它先对特征 z 拟合一个 k 分量的 GMM，然后用硬分配把每个特征分到最近的聚类中心，再用运行平均更新协方差矩阵，最后计算 CEM loss。这个 loss 跟分类任务的 loss 加权求和，一起反传。

---

## 第6页：我们的贡献

这就引出了我们的工作。GMM 用的是硬分配——每个特征只能属于一个聚类中心——这一步是不可微的，梯度需要近似。而且 GMM 对初始化和分量数 k 很敏感。

我们的方案是用 SlotAttention 替换 GMM。SlotAttention 用的是软注意力，全程可微，还有 GRU 迭代优化聚类中心。实验结果也证明了，我们的方法在 FaceScrub 上的 Dec Infer MSE 达到 0.0258，比论文最好的 0.0211 提升了 22.3%。

## 第8页：总览

在实现过程中，我发现原始代码有三个致命 bug。

第一个 bug，CEM 的 loss 方向完全反了——它在鼓励特征发散而不是聚拢。第二个 bug，方差是由一个神经网络来预测的，而不是几何计算出来的。第三个 bug，SlotAttention 是在单张图片内部做的聚类，而不是在同类别的多个样本之间做。

这三个 bug 加在一起，不仅让 CEM 完全无效，甚至可能让防御效果更差。

我的 cem-fixed 版本修复了这三个 bug，并且加了四个工程改进：FeatureMemoryBank、proj_down 降维、centroid detach、以及确定性推理。

---

## 第9页：架构对比图

左边是原始版本的流程，右边是我的 cem-fixed 版本。

原始版本：z 经过 unsqueeze 变成 [B, 1, D]，就是每张图片只有 1 个 token。然后过 SlotAttention——8 个 slot 去争抢 1 个 token，这完全没有意义。之后过 CrossAttention，再过一个 head_logvar 网络来预测方差，最后算 CEM loss——方向还是反的。

我的版本：z 先经过 proj_down 从 2048 降到 128 维，然后通过 MemoryBank 把同类别的多个样本拼在一起，SlotAttention 的输入变成 [1, N_c, 128]——N_c 个同类样本。8 个 slot 在这 N_c 个样本上做聚类，方差用几何公式直接算，loss 方向也是对的。

---

## 第10页：Bug 1 详解

Bug 1 是最致命的。看这两个公式。

原始公式是 ReLU(log θ - log v)。当方差 v 低于阈值 θ 时，loss 为正，所以它在惩罚低方差，驱动方差升高。这意味着 encoder 在把同类特征往外推——完全跟 CEM 的目标相反。

我修正后的公式是 ReLU(log v - log θ)。当方差 v 高于阈值 θ 时才惩罚，驱动方差降低，encoder 被迫把同类特征压缩到一起。

这个 bug 的影响非常严重：你越增大 lambd 权重，模型反而越帮攻击者。

---

## 第11页：Bug 2 详解

Bug 2 是方差的来源。原始代码用了一个 head_logvar 网络——就是一个 Linear 层加 Softplus——来预测方差。但问题是，这个网络有自己的可学习参数。优化器可以通过调整 head_logvar 的权重来满足 loss，完全不需要改 encoder。CEM 的压力被这个预测头吸收掉了，encoder 的特征完全不受约束。

我的做法是直接用几何公式计算方差：取 attention 权重作为软分配，乘以特征到聚类中心的距离平方。这样方差就是真实的欧式距离，没有任何可学习参数参与，梯度直接传到 encoder。

---

## 第12页：Bug 3 详解

Bug 3 是最大的语义错误。CEM 需要的是跨样本聚类——在同一类别的多个样本之间找子群。但原始代码做了一个 unsqueeze(1)，把每张图变成 [B, 1, D]，就是每张图只有 1 个 token。8 个 slot 去争 1 个 token，结果就是所有 slot 都坍缩到同一个点，完全无法做有意义的聚类。

我的修正是：先按类别筛选当前 batch 的特征，再从 MemoryBank 中取出历史特征，拼在一起。SlotAttention 的输入变成 [1, N_c, 128]——N_c 个同类样本。这样 8 个 slot 就是在 N_c 个样本上做竞争，真正在找类内的子群结构。

---

## 第13页：Fix 1 - MemoryBank

修了 Bug 3 之后有一个新问题：FaceScrub 有 530 个类别，batch size 是 256，平均每个类每个 batch 只出现不到 1 次。如果只靠当前 batch，大部分类会被完全跳过。

所以我设计了 FeatureMemoryBank——一个环形缓冲区，为每个类存 64 个历史特征。每个 step，先把当前 batch 的投影特征（无梯度）写入 bank，然后对每个类取出历史特征，跟当前 batch 的特征拼接。这样每个类的 N_c 就不再是 0 或 1，而是足够多的样本。

关键的梯度设计是：bank 中的历史特征没有梯度，只有当前 batch 的特征有梯度。所以 CEM loss 只会更新当前的 encoder 权重。

---

## 第14页：Fix 2 - proj_down

FaceScrub 的 encoder 输出是 [B, 8, 16, 16]，flatten 之后是 2048 维。如果直接在 2048 维上跑 SlotAttention，Q/K/V 矩阵就是 2048×2048，加上 GRU 和 MLP，总参数量超过 1 亿。

我加了一个 proj_down——一个简单的线性层，从 2048 投影到 128 维。这样 SlotAttention 的参数量从 1 亿降到 50 万，减少了 200 倍。而且梯度是完全保留的，因为线性投影是可微的，2048 个维度都能收到 CEM 的梯度信号。

---

## 第15页：Fix 3 & 4

Fix 3 是 centroid detach。计算方差时，聚类中心 c 是由 attention 权重和特征做加权平均得到的，也就是 c 对特征有梯度。如果不 detach，优化器有两条路：一条是让特征靠近中心（正确的），另一条是让中心靠近特征（偷懒的快捷方式）。加了 detach 之后，只剩一条路——特征必须真正地往中心聚拢。

Fix 4 是确定性推理。SlotAttention 初始化 slot 时用了随机采样 μ+σε。训练时这没问题，但 MIA 评估时每次 forward pass 给出不同的 attention 分布，结果不可复现。我改成了训练时用随机初始化，推理时只用 μ，确保评估结果完全可复现。

---

## 第16页：SlotAttention 机制详解

这一页详细讲 SlotAttention 的算法流程。对每个类 c：

1. 输入是 N_c 个特征向量，来自当前 batch 加 memory bank。
2. 初始化 S 个 slot（S 默认是 8），用可学习的 μ 加随机扰动。
3. 迭代 T 轮（T=3-5）：每轮先算 attention——slot 的 query 跟特征的 key 做点积再 softmax，然后按 attention 权重做加权平均得到更新向量，通过 GRU 更新 slot 状态，最后加一个 MLP 残差。

跟 GMM 的对比：GMM 是硬分配、运行平均更新、一次性分配；SlotAttention 是软分配、GRU 更新、多轮迭代细化。最关键的区别是 SlotAttention 全程可微，梯度是精确的。

---

## 第17页：完整训练管线

这一页展示了整个端到端的训练流程：

输入 64×64 的人脸图片 → VGG11 的前两层卷积加 LCA 层 → 得到 smashed data [B, 8, 16, 16] 加高斯噪声 → flatten 到 2048 维再 proj_down 到 128 维 → MemoryBank 按类别拼接历史特征 → SlotAttention 在同类样本上聚类 → 几何方差计算 → 最终 loss 是任务 loss 加 λ 乘以 CEM loss 再加 α 乘以距离相关 loss。

所有的梯度从 CEM loss 反传穿过 proj_down 一直到 encoder，encoder 被迫压缩同类特征。

## 第26页：为什么 SlotAttention 比 GMM 好

这一页总结了核心原因。

GMM 用硬分配，梯度需要近似，encoder 收到的更新信号是有噪声的、不精确的。协方差用运行平均更新，统计量是滞后的。初始化敏感，而且没有迭代细化。

SlotAttention 用软注意力，全程可微，encoder 收到的是平滑的、信息量丰富的梯度。GRU 做聚类中心更新，有学习到的动态。slot 的初始化是可学习的。而且有 T 步迭代细化，每一步都在改善聚类质量。

一句话总结：更平滑的梯度 → 更有效的 CEM 信号 → 更紧凑的子聚类 → 攻击者重建误差增加 22.3%。

## 可能被问到的问题和回答准备

**Q1：你的实验配置跟论文不完全一样（噪声更大、epoch 更多），这样比公平吗？**

A：论文发表的就是他们调到最好的结果。我们的方法不管用什么超参组合，MSE 都超过他们，说明是架构本身的优势。而且我们也准备了控制变量的实验（run_no.sh），固定所有基础参数，只换 CEM 机制，结果还在跑。

**Q2：准确率下降了 2.2%，这个可以接受吗？**

A：可以。论文中的 PATROL+CEM 准确率 79.88%，Dropout+CEM 是 79.19%，都在 78-80% 这个范围。在隐私防御领域，用少量准确率换取大幅隐私提升是标准做法。

**Q3：为什么 slot_dim=64 比 128 和 256 好？**

A：slot_dim=64 相当于一个更强的信息瓶颈。2048 维投影到 64 维时，特征被迫压缩得更紧凑，slot 之间的竞争更激烈，形成更紧凑的子聚类。

**Q4：SlotAttention 会增加多少计算开销？**

A：proj_down 后只有 0.5M 参数（主模型 9.8M），计算开销可以忽略不计。论文 S.6 也确认 CEM 不引入额外开销。
